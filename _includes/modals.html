Find Enchiladas

WHAT: Personalized Food Scraper and Scheduled SMS Notifier
HOW: Laravel 5, MySQL 5, Twilio API, HTML, CSS, JavaScript, Cron

Find Enchiladas is my first web development project. Originally created as part of a Distributive Software Architecture class project, we decided to continue developing the application.

Find Enchiladas pulls dining hall menus from 7 different dining halls and compares for matches based off someone's favorite foods. But we didn't stop there. As college students, we want information delivered to our fingertips. To accomodate that, we use the Twilio messaging API to send regular text messages using Cron Jobs.

My project partner, Andrew Atwong, and I recently improved the relational database and pushed this project into Laravel to better consolidate code and improve its efficiency. The app features a signup page, encrypted passwords, forgotten password retrievals, and settings to turn off notifications. At some point, we plan to include further improvements such as nuanced preferences, better searches, and maybe even mobile apps.

  • Client: The Claremont Colleges
  • Date: March 2015
  • Service: Web Development

Official GHCHS Speech & Debate Team Site

WHAT: Continuosuly Updated Team Newsfeed and Custom Content for Admins
HOW: Instagram, Twitter, Laravel 5, Instagram API, Twitter API, FullCalendar.js, MySQL 5

The Granada Hills Speech and Debate Team Newsfeed was created at the request of current members of the GHCHS Speech and Debate Team. The team originally used social media outlets like Facebook to keep students informed regarding event information.

To fix this, I developed a relational database and Laravel 5 web application to operate a newsfeed that pulls data from Instagram and Twitter admin accounts. This functions off a cron that frequently searches for new posts and allows designated admins (team officers) to approve or deny the social media post. The site also features a calendar, and upcoming events section where an admin can post new info after securely logging in.

  • Client: Granada Hills Speech and Debate Team
  • Date: August 2015
  • Service: Web Development

LinkedIn Non-Technical HR Hackathon - 2nd Place

WHAT: Competition to Ideate an HR Tool to increase Employee Happiness in the Workplace
HOW: Photoshop, Powerpoint, Illustrator, HTML, CSS, JavaScript, Bootstrap

The LinkedIn Human Resources Hackathon was an exciting and interesting experience. Being a non-technical hackathon, I had to pivot and accommodate the group without using programming. Hackers were instructed to think of an application or solution to the age-old problem residing in human resources: why are people not interested in their work?

We decided to pitch the Impact Dashboard which would agglomerate data from a company's pool of data and crunch numbers to provide personal statistics to each employee. Such an board would ideally boost the morale of employees through tangible and real statistics that showed positive impact through their work.

My primary goal was to create a dashboard that represented this. Using Adobe Photoshop CC and a Bootstrapped HTML template, I was able to create a very colorful mockup that captured our idea using a LinkedIn color scheme. This visual along with spectacular presentations helped us reach the finals and then 2nd place overall.

  • Client: LinkedIn Headquarters
  • Date: August 2015
  • Service: Graphic Design/Ideation

Code For America - Products Page

WHAT: Redesigned Code For America Products Page to better showcase CFA Products
HOW: Jekyll, Liquid, HTML, CSS, Isotope

The Products Page was part of a Code For America initiative to redesign the main site in order to reflect new goals and missions for the company. We were told that the desired audience was younger government officials who wanted to create change. In order to showcase products, we had to organize the apps efficiently and effectively so users could search through the page quickly.

The Products Page was developed using Jekyll and Liquid. We decided to use isotope to organize apps using button clicks. We categorized apps by four sections (Economic Development, Communication and Engagement, Safety and Justice, and Health) and subdivided those into stages (Betas, Alphas, Live, etc). Ultimately, we created a site that had quite a bit of functionality that operated off a small CSV file exported from an organized Google Doc.

After a few user testing sessions, we began to understand that the page needed more work on the high level, (i.e. how we categorize and sort the apps). But people appreciated things like a featured app, and the ability to categorize. However, the site will certainly take more time before it can be put live.

Update: Code For America made modifications to our experimental page but they finally hosted the page!

  • Client: Code For America
  • Date: July - August 2015
  • Service: Front End Development

Code For America - Data Visualization

WHAT: Summit Data Visualization App
HOW: Node.js, Google Sheets, HTML, CSS, Charts.js, D3, Underscore

The Summit Registrants App was a small project that sought to represent an accumulation of Summit data online. Code For America usually runs these pages throughout TV monitors around the office.

Having never worked with Node.js in the past, I had to really push myself to understand what was happening in this code. Ultimately, I was able to understand that and the various JS libraries to make the application work. From there, it boiled down to creating computations based of available data. Whether it was calculating how many different types of people were going, or total funds obtained from ticket sales, all the data was taken dynamically from a Google Sheet.

That being said, the challenging part of this project was synchronizing the data with a Google Sheet constantly being updated from a ticket registration form. Once we figured out how to obtain an API key for our specific Google Sheet, we had the app download new data in 5 minute intervals.

  • Client: Code For America
  • Date: August 2015
  • Service: Web Development

Wow Beauty Supply - Remodel

WHAT: E-Commerce Website Makeover, Migration, and Maintenance
HOW: Shopify, Liquid, CSS

Wow Beauty Supply is a local beauty supply store that was looking for a way to redesign their website. Originally Wow Beauty used Magento to manage and maintain their store, but it fell into disarray because Magento was complicated and confusing to use. After doing my research and discussions with others, I recommended and managed the store's transition from Magento to Shopify.

Along the way, I also transferred their domain to a more reliable service provider, migrated their inventory info, and built up a new website after discussing Shopify templates.

Currently, we are updating the images, and product descriptions, but the site has been successfully transferred and will soon be ready to go.
