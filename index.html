    Find Enchiladas

    TL;DR: Favorite Food Scraper using Laravel 5, MySQL 5, Twilio API, HTML, CSS, JavaScript

    Find Enchiladas is my first web development project. Originally created as part of a Distributive Software Architecture class project, we decided to continue developing the application.

    The goal of Find Enchiladas was to search 7 different dining hall menus based off someone's favorite foods. But, we wanted something more. As college students, it's hard to check the menu everyday when we are running across campus. To accomodate that, we implemented the Twilio messaging API to send text messages using a Cron.

    My project partner, Andrew Atwong, and I recently put this project into Laravel to better consolidate and organize our code. At some point, we plan to include further improvements such as nuanced preferences, better searches, and maybe even mobile apps.

    • Client: The Claremont Colleges
    • Date: March 2015
    • Service: Web Development

    Claremont Rise

    TL;DR: Automated Emailer for College News & Event Scraper using Laravel 5, Goutte, MySQL 5, Sendgrid, Forecast.io, Cron, HTML, CSS, JS

    My original goal for Claremont Rise was strictly educational; how much stuff could I learn in one project? I wanted to understand Laravel to go through the process of building a dynamic website from the ground up. This included planning out the relational database, creating a logo, and also understanding my users (college students, professors, deans, etc). The first iteration of Claremont Rise attempted to combine information from across the 5C's. This included information about campus events, campus news, various student newspapers, special talks at the Ath, weather, and even a daily campus instagram picture. A sample of it can be seen below.

    However, after getting back to college, Claremont Rise became extremely hard to maintain and other campus organizations began offering similar services. Consequently, I decided to narrow my focus to solely news organizations across the 5Cs. Despite this, I realized that there are over 10 news sources on campus.

    • Client: Claremont Colleges

    Official GHCHS Speech & Debate Team Site

    TL;DR: Instagram, Twitter, Team-Info Newsfeed using Laravel 5, Instagram API, Twitter API, FullCalendar, MySQL 5

    The Granada Hills Speech and Debate Team Newsfeed was created at the request of current members of the GHCHS Speech and Debate Team. Originally the team used social media outlets like Facebook to keep students informed regarding event information.

    Additionally, the team had no central organization for team updates during/after competitions which limited engagement.

    To fix this, we implemented a newsfeed that pulls new data from Instagram and Twitter directly from admin accounts, and also creates pending images if a person was to use the #ghsnd. I created a cron that consistently searches for new posts and allows designated admins (team officers) to approve or deny the social media post. The site also features a calendar, and upcoming events section where new info can be posted via an admin page secured by a login form.

    • Client: Granada Hills Speech and Debate Team
    • Date: August 2015
    • Service: Web Development

    Code For America - Products Page

    TL;DR: Redesigned Code For America Products Page using Jekyll, Liquid, HTML, CSS, Isotope

    The Products Page was part of a Code For America initiative to redesign the main site in order to reflect new goals and missions for the company. We were told that the desired audience was younger government officials who wanted to create change. In order to showcase products, we had to organize the apps efficiently and effectively so users could search through the page quickly.

    At some point, we thought about utilizing isotope to organize apps using button clicks. We categorized apps by four sections (Economic Development, Communication and Engagement, Safety and Justice, and Health) and subdivided those into stages (Betas, Alphas, Live, etc). Ultimately, we created a site that had quite a bit of functionality that operated off a small CSV file exported from an organized Google Doc.

    After a few user testing sessions, we began to understand that the page needed more work on the high level, (i.e. how we categorize and sort the apps). But people appreciated things like a featured app, and the ability to categorize. However, the site will certainly take more time before it can be put live.

    • Client: Code For America
    • Date: July - August 2015
    • Service: Front End Development

    Code For America - Data Visualization

    TL;DR: Summit Data Visualization App using Node.js, Google Sheets, HTML, CSS, Charts.js, D3, Underscore

    The Summit Registrants App was a small project that sought to represent an accumulation of Summit data online. Code For America usually runs these pages throughout TV monitors around the office.

    Having never worked with Node.js in the past, I had to really push myself to understand what was happening in this code. Ultimately, I was able to understand that and the various JS libraries to make the application work. From there, it became computations that I pulled from the data. Whether it was calculating how many different types of people were going via a pie chart, or total funds obtained from ticket sales, all the data was avaiable in the Google Sheets.

    A challenging part of this project was synchronizing the data with a constantly updating Google Sheet that was tied to the ticket registration forms. Once we figured out how to obtain an API key for our specific Google Sheet, we had the app download new data in 5 minute intervals.

    • Client: Code For America
    • Date: August 2015
    • Service: Web Development

    Wow Beauty Supply - Remodel

    TL;DR: E-Commerce Website Makeover and Migration - Shopify, Liquid, CSS

    Wow Beauty Supply is a local beauty supply store that was looking for a way to redesign their website. Originally Wow Beauty used Magento to manage and maintain their store, but it fell into disarray because Magento was complicated and confusing to use. After doing my research and discussions with others, I recommended and managed the store's transition from Magento to Shopify.

    Along the way, I also transferred their domain to a more reliable service provider, migrated their inventory info, and built up a new website after discussing Shopify templates.

    Currently, we are updating the images, and product descriptions, but the site has been successfully transferred and will soon be ready to go.

    • Client: Wow Beauty Supply
    • Date: August 2015
    • Service: Web Development
